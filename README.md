# HTML-Weaver 🕸️

> 新一代 AI 原生 HTML 编辑器
> **进可替代 PPT/Word，退可作为现代 Dreamweaver**
> 核心思想：**改参数，不改代码**

---

## 为什么 HTML 难改？

**痛点**：领导的修改通常集中在标题、数字、模块顺序等，如果这些内容混在 HTML 代码里，每次修改都像重写一遍，效率极低。

**核心问题**：我们没有预先将"哪些东西以后一定会改"显性化。

---

## 解决方案："改参数，不是改代码"

真正好改的 HTML，关键在于把页面拆分为**"页面壳 + 配置 + 数据"**的组合：

- **识别高频变化项**：标题、重点数字、模块显隐、布局选项
- **抽成配置**：集中管理，不散落在代码角落
- **页面只负责渲染**：配置改变，内容自动更新
- **保护复杂逻辑**：核心计算和联动关系不应在汇报现场临时修改

---

## 三层架构

```
┌─────────────────────────────────────────────┐
│           可视化编辑层 (编辑器)               │
│   • 大纲树 (幕布式缩进流)                    │
│   • Block 语义化 (/hero, /gallery)          │
│   • 配置面板 (参数化编辑)                    │
│   • 实时预览                                 │
├─────────────────────────────────────────────┤
│           中间抽象层 (JSON Schema)           │
│   • 结构定义 (Section/Block/Field)           │
│   • 配置分离 (config.js)                     │
│   • 版本管理 (config_v1.js, v2.js...)        │
├─────────────────────────────────────────────┤
│           渲染输出层 (HTML/CSS/JS)           │
│   • 页面壳 (index.html)                      │
│   • 主题系统 (响应式布局)                    │
│   • 交互逻辑                                 │
└─────────────────────────────────────────────┘
```

## 核心特性

### 1. 大纲树编辑器 (继承幕布优点)

**无限层级 + Tab/Enter 流式构建**：
```
[Hero] ▼
  Title: 可编辑文本
  Subtitle: 可编辑文本
[Features] ▼
  [Item 1]
  [Item 2]
[Gallery] ▼
  Layout: 3列
  Images: [...]
```

- **Tab/Enter/Shift+Tab**：快速调整层级
- **语义 Block**：`/hero` 创建 Hero Block，`/gallery` 创建 Gallery
- **三级折叠**：Section → Block → Field

### 2. 配置面板 (核心创新)

选中任意 Block，右侧显示可配置参数：

| 参数类型 | 示例 |
|---------|------|
| 文本 | 标题、副标题、按钮文字 |
| 数字 | 百分比、统计数据、价格 |
| 布局 | 2列/3列/轮播/网格 |
| 主题 | 色彩方案、字体、间距 |
| 布尔 | 模块显隐、动画开关 |

**修改即时生效**：调整参数 → 预览实时更新

### 3. 配置版本管理

```
reports/
├── index.html        # 页面壳
├── config.js         # 当前配置（软链接）
├── config_v1.js      # 版本1：初稿
├── config_v2.js      # 版本2：领导修改1
├── config_final.js   # 版本3：最终版
├── data.js           # 业务数据
└── theme.css         # 视觉主题
```

- 每次修改保存为新版本，不像 Word 一样覆盖
- 可随时回滚到任意版本
- 支持 A/B 测试不同配置

### 4. 模板画廊

参考 [io3000.com](https://io3000.com) 的展示方式：

- **按色彩**：冷色/暖色/黑白/渐变
- **按主题**：汇报/个人主页/产品展示/简历
- **按用途**：PPT 替代/网页/Landing Page

点击模板 = 加载默认配置，立即开始编辑

## 文件结构规范

一个可维护的 HTML 报告由以下文件组成：

```
project/

```

## 技术栈

- **前端**：React + TypeScript

- **配置**：JSON Schema 定义结构

## 使用流程

### 场景：制作季度汇报

1. **选择模板** → "汇报模板 - 蓝色商务风"
2. **编辑大纲** → Tab/Enter 调整结构
3. **修改内容** → 点击 Hero，右侧改标题和数字
4. **实时预览** → 看到最终效果
5. **保存版本** → config_v1.js（初稿）
6. **领导修改** → 直接调参数，另存 config_v2.js
7. **导出部署** → 生成独立 HTML，上传到 GitHub Pages

## 核心设计决策

**Q: 为什么用 JSON Schema 而不是直接编辑 HTML？**

A: 保护复杂逻辑，只暴露可变的参数；支持版本回滚；非技术人员也能修改。

**Q: Block 和纯文本大纲的区别？**

A: Block 有类型，知道如何渲染自己；有可配置参数；可以有子 Block。

**Q: 配置分离的好处？**

A: 修改标题不需要动代码；可以 A/B 测试；团队协作只合并配置，不冲突。
